## 1. 产品概述

### 1.1 产品定位

Book Record 是一款面向**个人阅读爱好者**的读书笔记管理软件，帮助用户记录阅读感悟、收藏精彩金句、追踪阅读进度，打造个人的阅读知识库。

### 1.2 目标用户

- **主要用户**：个人阅读爱好者
- **使用场景**：
  - 记录阅读过程中的感悟和思考
  - 收藏书籍中的精彩语句
  - 追踪阅读进度和习惯
  - 整理和回顾阅读收获

### 1.3 核心价值

- 📝 **深度记录**：块级富文本编辑，支持 Markdown，让感悟记录更自由
- 🔍 **智能检索**：全文检索能力，快速定位任何内容
- 📊 **数据洞察**：高级阅读分析，了解阅读习惯
- 🔗 **便捷分享**：一键分享感悟和金句

## 2. 用户故事与验收标准

### 2.1 用户认证模块

#### US-001 用户注册

**作为** 一个新用户
**我想要** 创建一个账户
**以便于** 使用系统的所有功能

**验收标准**：
- [ ] 用户可以输入用户名、邮箱、密码进行注册
- [ ] 用户名长度 3-20 字符，仅允许字母、数字、下划线
- [ ] 邮箱格式验证正确
- [ ] 密码长度至少 8 位，包含字母和数字
- [ ] 用户名和邮箱不可重复
- [ ] 注册成功后自动登录并跳转到首页
- [ ] 注册失败时显示具体错误信息

---

#### US-002 用户登录

**作为** 一个已注册用户
**我想要** 登录系统
**以便于** 访问我的个人数据

**验收标准**：
- [ ] 用户可以使用用户名/邮箱和密码登录
- [ ] 登录成功后获取 JWT Token
- [ ] Token 有效期内无需重复登录
- [ ] 登录失败显示明确的错误提示
- [ ] 支持"记住我"功能延长 Token 有效期

---

#### US-003 用户登出

**作为** 一个已登录用户
**我想要** 安全登出系统
**以便于** 保护我的账户安全

**验收标准**：
- [ ] 点击登出按钮清除本地 Token
- [ ] 登出后跳转到登录页
- [ ] 登出后无法访问需要认证的页面

## 3. 非功能性需求

### 3.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | 首屏 < 3秒 |
| API 响应时间 | < 500ms |
| 列表加载 | 支持分页，单页 ≤ 20 条 |
| 搜索响应 | < 1秒 |

### 3.2 安全要求

- 所有 API（除认证接口）需 JWT 认证
- 密码使用 BCrypt 加密存储
- 生产环境 JWT 密钥通过环境变量配置
- 用户数据完全隔离，禁止跨用户访问
- 敏感操作记录日志

### 3.3 可用性要求

- 系统可用性 ≥ 99%
- 支持数据自动备份
- 支持数据恢复

### 3.4 兼容性要求

- 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 服务器：Linux/Windows
- 数据库：H2（开发）、PostgreSQL（生产）

## 7. 术语定义

| 术语 | 定义 |
|------|------|
| 感悟 | 用户对书籍内容的思考、总结、疑问或洞察记录 |
| 金句 | 用户从书籍中摘录的精彩语句 |
| 书架 | 用户所有书籍的集合展示 |
| 块级编辑 | 以段落、标题、列表等为基本单位的富文本编辑方式 |
| 全文检索 | 对所有文本内容进行关键词搜索的能力 |
